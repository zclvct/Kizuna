## 测试场景

### 测试 1: 第一次启动引导对话

**目的**: 验证第一次启动时的引导对话流程

**步骤**:
1. 删除 `data/character.json`（如果存在）
2. 运行应用：`python run.py`
3. **预期结果**:
   - 对话窗口自动显示
   - 角色主动显示问候语（随机选择）
   - 问候语类似："你好，初次见面！我还没有名字，能给我起个名字吗？"

4. 回复角色，例如："以后你就叫小秘"
5. **预期结果**:
   - 角色调用 edit_persona 工具
   - 角色回复："好的，主人！以后我就是小秘了~的说 ♪"
   - 角色继续询问更多信息

6. 回复："叫我小明就好"
7. **预期结果**:
   - 角色调用 edit_persona 工具记录
   - 角色回复："好的，小明！很高兴认识你 😊"

8. 检查 `data/character.json`：
   ```json
   {
     "name": "小秘",
     "second_person": "小明",
     ...
   }
   ```

**通过标准**:
- [ ] 第一次启动自动显示对话窗口
- [ ] 角色主动问候并询问名字
- [ ] 角色使用 edit_persona 工具保存信息
- [ ] character.json 正确保存信息

### 测试 2: 设置窗口 - 角色设定编辑

**目的**: 验证角色设定编辑功能

**步骤**:

2. 切换到"角色设定"标签页
3. **预期结果**:
   - 显示当前角色设定的所有字段
   - 名字、性别、年龄、生日等
   - 性格、说话风格、自称、称呼
   - 关系、用户昵称
   - 背景故事
   - 喜欢、讨厌

4. 修改一些字段：
   - 名字改为"小秘酱"
   - 性格改为"活泼开朗，非常傲娇"
   - 喜欢改为"主人, 咖啡, 可爱的东西"
5. 点击"保存"
6. **预期结果**:
   - 设置窗口关闭
   - 角色设定更新

7. 重新打开设置窗口，验证修改已保存
8. 在对话中输入："自我介绍一下"
9. **预期结果**:
   - 角色根据新设定进行介绍
   - 名字应为"小秘酱"
   - 性格描述应更新

**通过标准**:
- [ ] 设置窗口正确加载角色设定
- [ ] 可以修改所有字段
- [ ] 修改后正确保存到 character.json
- [ ] 对话中角色使用新设定

### 测试 3: 对话中学习用户信息

**目的**: 验证角色在对话中学习用户信息的能力

**步骤**:
1. 在对话中输入："我喜欢喝咖啡"
2. **预期结果**:

4. **预期结果**:
   - 角色记录作息信息
   - 例如："好的，我会记住小明的作息时间~"

5. 打开 `data/character.json`，检查 learned_facts：
   ```json
   {
     "learned_facts": {
       "用户喜好": "咖啡",
       "用户作息": "每天晚上12点睡觉"
     }
   }
   ```

6. 输入："我讨厌吃辣"
7. **预期结果**:
   - 角色记录讨厌的事物

8. 检查 character.json 中的 dislikes

**通过标准**:
- [ ] 角色在对话中记录用户信息
- [ ] 使用 edit_persona 工具（add_fact）
- [ ] learned_facts 正确保存

### 测试 5: 敏感字段修改确认

   - 在角色询问确认时，输入其他内容

**通过标准**:
- [ ] 敏感字段修改需要确认

## 常见问题排查

### 问题 1: 第一次启动没有显示引导对话
**排查**:
- 检查 `data/character.json` 是否已存在
- 检查 `CharacterPersona.is_first_run()` 返回值
- 检查日志中是否有"第一次启动，显示引导对话"

### 问题 2: 角色没有使用 edit_persona 工具
**排查**:
- 检查 LLM 是否返回 tool_calls
- 检查 Function Calling 配置是否正确
- 检查工具定义是否正确

### 问题 3: 设置没有保存到文件
**排查**:
- 检查 `data/` 目录是否存在
- 检查文件权限
- 检查 `CharacterManager.save()` 是否被调用

### 问题 4: System Prompt 没有更新
**排查**:
- 检查 `LLMClient._build_system_prompt()` 是否读取最新设定
- 检查是否需要重启应用
