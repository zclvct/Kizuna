# Phase 7 测试指南

**版本**: 1.0
**日期**: 2026-03-20
**测试范围**: 整合与完善功能

---

## 测试准备

### 环境要求

- Python 3.10+
- 已安装所有依赖（`pip install -r requirements.txt`）
- 至少配置了一个 LLM（OpenAI/Anthropic/Ollama）
- 可选：准备了 Live2D 模型

### 测试数据

1. **LLM API Key**
   - OpenAI API Key 或 Anthropic API Key
   - 或本地 Ollama 服务运行

2. **Live2D 模型**（可选）
   - 将模型放入 `assets/live2d/` 目录

3. **测试用定时任务**
   - 准备几个测试用的 Cron 表达式

---

## 测试用例

### TC1: 设置窗口 - LLM 标签页

**测试目标**: 验证 LLM 配置功能

**前置条件**: 应用已启动

**测试步骤**:
1. 右键点击角色 → 选择"设置"
2. 切换到"LLM"标签页
3. 修改提供商为不同的选项
4. 修改模型名称
5. 输入/修改 API Key
6. 输入/修改 Base URL
7. 点击"保存"按钮
8. 重启应用
9. 重新打开设置窗口，验证配置是否持久化

**预期结果**:
- 所有输入框可正常输入
- 保存后配置正确保存到 `data/config.json`
- 重启后配置正确加载
- 对话功能可正常使用配置的 LLM

---

### TC2: 设置窗口 - Live2D 标签页

**测试目标**: 验证 Live2D 配置功能

**前置条件**: 应用已启动，已准备 Live2D 模型

**测试步骤**:
1. 打开设置窗口
2. 切换到"Live2D"标签页
3. 点击"浏览"按钮，选择 Live2D 模型目录
4. 调整模型缩放比例（0.5 - 2.0）
5. 启用/禁用心情动作
6. 启用/禁用空闲动作
7. 设置空闲动作间隔（10-60 秒）
8. 选择测试心情，点击"测试心情动作"
9. 选择测试意图，点击"测试意图动作"
10. 点击"保存"按钮
11. 验证 Live2D 显示和动作

**预期结果**:
- 模型路径选择对话框正常显示
- 模型缩放实时生效
- 心情动作和空闲动作开关正常工作
- 空闲动作间隔可正确设置
- 测试按钮能触发对应动作
- 保存后配置持久化
- Live2D 正常显示和播放动作
