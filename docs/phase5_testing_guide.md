## 测试场景

### 测试 1: 心情驱动动作

**目的**: 验证 LLM 根据对话内容选择合适的动作

**步骤**:
1. 确保"心情驱动动作"已启用（默认启用）
2. 在对话中输入："今天发工资了！超开心！"
3. **预期结果**:
   - LLM 调用 `play_motion` 工具
   - 参数：`mood="happy"`, `new_mood="happy"`
   - Live2D 播放开心动作（如 happy_01 或 happy_02）
   - 角色回复表现出开心

4. 输入："我很难过，考试没考好"
5. **预期结果**:
   - LLM 调用 `play_motion` 工具
   - 参数：`mood="sad"`, `new_mood="sad"`
   - Live2D 播放难过动作（如 sad_01）
   - 角色安慰用户

6. 输入："你说我今天穿什么好呢？"
7. **预期结果**:
   - LLM 调用 `play_motion` 工具
   - 参数：`mood="thinking"`
   - Live2D 播放思考动作（如 thinking_01）

**通过标准**:
- [ ] LLM 正确调用 play_motion 工具
- [ ] 参数正确（mood 和 new_mood）
- [ ] Live2D 显示对应动作
- [ ] 角色心情状态正确更新

### 测试 3: 空闲动作定时器

**目的**: 验证空闲时自动播放动作

**步骤**:
1. 打开设置窗口 → Live2D
2. 确认"启用空闲动作"已勾选
3. 设置空闲间隔为 10 秒（便于测试）
4. 保存设置
5. **预期结果**:
   - 空闲时（不进行对话）每 10 秒播放一次空闲动作
   - 动作从 idle_motions 列表中随机选择

6. 等待 30 秒，观察 Live2D
7. **预期结果**:
   - 至少播放 3 次空闲动作
   - 每次动作可能不同（随机选择）

8. 取消勾选"启用空闲动作"
9. 保存
10. 等待 30 秒
11. **预期结果**:
    - 不再播放空闲动作

**通过标准**:
- [ ] 空闲动作按设定间隔播放
- [ ] 可以启用/禁用空闲动作
- [ ] 间隔时间正确

## 常见问题排查

### 问题 1: 动作没有播放
**排查**:
- 检查"启用对话时动作"是否勾选
- 检查 LLM 是否调用 play_motion 工具
- 检查日志中是否有错误
- 检查 motions.json 配置是否正确

### 问题 2: 空闲动作不播放
**排查**:
- 检查"启用空闲动作"是否勾选
- 检查空闲间隔设置是否合理
- 检查 Live2DWidget 是否正常初始化
- 检查计时器是否启动

### 问题 3: 心情状态不更新
**排查**:
- 检查 play_motion 的 new_mood 参数
- 检查 CharacterManager 的 state 对象
- 检查 _update_mood_display() 是否被调用

### 问题 4: 测试按钮不工作
**排查**:
- 检查 MotionController 是否正确初始化
- 检查配置是否正确加载
- 检查是否绑定了正确的回调函数

### 问题 5: 动作配置没有保存
**排查**:
- 检查 `data/` 目录是否存在
- 检查文件权限
- 检查 _save_config() 是否被调用
- 检查 motions.json 是否正确写入
